## 三、每日工作流程

### 08:00~08:30 Claude生成分析

```
输入：  ETF历史行情（新浪API）+ 财经新闻（Playwright）+ 盘前checklist
输出：  ETF日报_YYYYMMDD.md（包含分析 + SIGNAL字典）
        signal_YYYYMMDD.json（可选，备用）
```

报告结构：
```
一、昨日市场复盘（ETF涨跌表）
二、盘前预判与操作策略（各板块涨跌判断）
三、今日关键事件/宏观日历
四、持仓策略（核心-卫星配置）
五、风险监控
六、SIGNAL 代码块（可复制粘贴到聚宽）
```

### 08:30~08:45 用户执行

```
① 打开报告 → 审阅分析
② 找到底部的 SIGNAL = {...} 代码块
③ 复制 → 打开聚宽策略 → 覆盖顶部 SIGNAL 区域 → 保存
④ 全程约 15 秒
```

### 09:30 策略自动运行

```
① 读取 SIGNAL 字典
② 信号 → 权重映射：1=×1.2, 0=×1.0, -1=×0.5
③ 归一化，扣除现金比例（cash_pct）
④ 对比当前持仓，偏差>2%时调仓
⑤ 若SIGNAL为空 → 自动使用均线备选信号
```

### 盘中（可选） 如需修正

```
如果盘中出现重大偏差：
  → 重新生成SIGNAL信号
  → 用户更新聚宽策略
  → 第二天自动执行
```

### 4.2 策略核心逻辑

```python
# 每天只需要更新这个 SIGNAL 字典
SIGNAL = {
    "date": "2026-07-15",
    "market_phase": "防御期",
    "signals": {
        "510300.XSHG": 0,   # -1卖出 0持有 1买入
        "510880.XSHG": 1,
        "512800.XSHG": 1,
        "588000.XSHG": -1,
        "512170.XSHG": 1,
    },
    "cash_pct": 0.10,
}

# 信号→权重映射
SIGNAL_MULT = {1: 1.2, 0: 1.0, -1: 0.5}

# 每日09:30自动调仓，偏离>2%触发交易
```

### 4.3 关键参数

| 参数 | 值 | 说明 |
|:----:|:---:|------|
| 调仓频率 | 每天检查 | run_daily，有信号偏差>2%才调 |
| 偏离阈值 | 2% | 偏差超过2%才调仓，避免频繁交易 |
| 单只上限 | 35% | 任何ETF不超过35%仓位 |
| 备选策略 | 均线MA5/MA20 | 无SIGNAL时自动启用 |

## 七、关键经验教训（防止再犯）

| # | 教训 | 来源 |
|:-:|------|:----:|
| 1 | 盘前必查**新股/IPO日历**（`ak.stock_ipo_declare_em()`） | 7/15遗漏长鑫IPO |
| 2 | 当日经济数据发布时间到官方确认，不靠记忆 | 7/15搞错发布时间的教训 |
| 3 | 超跌>6%的品种即使看空也要考虑技术性反弹可能 | 7/15游戏误判 |
| 4 | 聚宽 `sum(dict.values())` 在特定Python版本会返回`dict_values`，必须用手动for循环累加 | 多次回测失败 |
| 5 | 聚宽 `get_price()` 取多标的时不能设 `skip_paused=True` | 回测失败 |
| 6 | 聚宽 `attribute_history()` 逐个取比 `get_price()` 批量取更稳定 | 多次比较后的结论 |
| 7 | ETF策略做纯技术指标回测效果差，必须结合宏观判断 | v7 vs v8回测对比 |
