## 三、每日工作流程

### 08:00~08:30 Claude生成分析

```
输入：  ETF历史行情（新浪API）+ 财经新闻（Playwright）+ 盘前checklist
输出：  ETF日报_YYYYMMDD.md（包含分析 + SIGNAL字典）
        signal_YYYYMMDD.json（可选，备用）
```

报告结构：
```
一、昨日市场复盘（ETF涨跌表）
二、盘前预判与操作策略（各板块涨跌判断）
三、今日关键事件/宏观日历
四、持仓策略（核心-卫星配置）
五、风险监控
六、SIGNAL 代码块（可复制粘贴到聚宽）
```

### 08:30~08:45 用户执行

```
① 打开报告 → 审阅分析
② 找到底部的 SIGNAL = {...} 代码块
③ 复制 → 打开聚宽策略 → 覆盖顶部 SIGNAL 区域 → 保存
④ 全程约 15 秒
```

### 09:30 策略自动运行

```
① 读取 SIGNAL 字典
② 信号 → 权重映射：1=×1.2, 0=×1.0, -1=×0.5
③ 归一化，扣除现金比例（cash_pct）
④ 对比当前持仓，偏差>2%时调仓
⑤ 若SIGNAL为空 → 自动使用均线备选信号
```

### 盘中（可选） 如需修正

```
如果盘中出现重大偏差：
  → 重新生成SIGNAL信号
  → 用户更新聚宽策略
  → 第二天自动执行
```

## 八、多策略架构（优化方向一）

### 8.1 多策略不会混淆判断——反而更清晰

> **核心原则：多策略 = 多个独立视角 → 投票汇总 → 综合决策**

每个策略相当于一个**独立的分析师**，从不同角度观察市场：

```
策略A（宏观驱动）→ Claude看地缘/政策 → 建议减仓芯片
策略B（动量跟踪）→ 看价格趋势         → 建议持有芯片（趋势未破）
策略C（估值/资金）→ 看估值分位+资金流  → 建议买入芯片（低估）

最终决策：
  如果A+B+C都看多 → 强烈买入
  如果A看空+B看多+C看空 → 分歧，减仓观望
  这是"3个分析师投票"，不是"1个人精神分裂"
```

**多策略不会让你混乱，反而让你知道"当前市场的共识是什么"。**

### 8.2 建议的三策略架构

```
┌─────────────────────────────────────────────────────────────┐
│                    组合决策引擎                              │
│  三策略投票加权 → 最终SIGNAL                                │
└──────────┬──────────────┬──────────────┬───────────────────┘
           │              │              │
    ┌──────┴──────┐ ┌─────┴──────┐ ┌───┴────────┐
    │ 主策略 70%  │ │ 技术策略    │ │ 资金流策略  │
    │             │ │ 20%        │ │ 10%        │
    ├─────────────┤ ├────────────┤ ├────────────┤
    │ Claude宏观  │ │ 均线+动量  │ │ 北向资金   │
    │ 分析驱动    │ │ 自动执行    │ │ +大单流向  │
    │             │ │            │ │            │
    │ 优势：判断  │ │ 优势：     │ │ 优势：     │
    │ 地缘/政策   │ │ 无情绪     │ │ 客观数据   │
    │             │ │ 无延迟     │ │           │
    │ 劣势：依赖  │ │ 劣势：     │ │ 劣势：     │
    │ Claude输出  │ │ 滞后指标   │ │ 信号噪     │
    └─────────────┘ └────────────┘ └────────────┘
```

### 8.3 三策略如何协同而不是冲突

```
当三种策略都指向同一方向时：
  → 满仓操作，置信度最高

当主策略和技术策略冲突时：
  → 主策略权重70%，技术策略20%
  → 最终信号偏向主策略，但减少仓位
  → 例如：Claude说买入(70%) + 均线说卖出(20%)
  → 综合：轻仓买入，而不是全仓

当三种策略严重分歧时：
  → 提高现金比例，等方向明确
  → 分歧本身就是最有价值的信息
```

### 8.4 实现方式

不需要修改每天的SIGNAL输入方式。三策略整合在**策略内部自动完成**：

```python
# 每天早上策略自动执行：
# 1. 读取 Claude 的 SIGNAL（主策略）
# 2. 计算均线/动量信号（技术策略）
# 3. 检查北向资金/大单流向（资金流策略）
# 4. 三策略投票加权 → 最终权重
# 5. 执行调仓
```

你每天的操作不变——仍然是复制SIGNAL到聚宽策略。**多策略整合由代码自动完成。**

## 十、更好的框架参考

### 10.1 有没有更好的"skill"可以参考？

我们的 etf-analysis + etf-operation 双Skill已经是**适合"人机协作"场景的框架**。市面上的量化框架都是**纯代码框架**，不适合Claude这种AI+人的模式：

| 框架 | 适合场景 | 不适合我们的原因 |
|:----:|:--------|:----------------|
| **vn.py** | 纯量化策略开发 | 需要编程，无法融入Claude的宏观分析 |
| **backtrader** | 纯回测 | 只能回测技术指标，无法处理地缘政治判断 |
| **聚宽/米筐** | 云端量化 | 和我们现在用的方式一致，没有额外优势 |
| **我们的双Skill** | **人+AI协作** | ✅ **专门为Claude分析+人工确认+自动执行设计** |

### 10.2 不建议改成纯量化框架

我们的核心竞争力是**Claude的宏观判断**，而不是代码跑得有多快。如果把精力花在把策略改成纯量化框架（vn.py/backtrader），反而丢掉了我们的优势。

**改进的正确方向**不是换框架，而是在现有框架上：
1. 增加技术策略作为辅助（多策略投票）
2. 提升数据质量（baostock）
3. 积累回测数据验证Claude准确率
