## 一、体系定位

### 1.1 与 `etf-analysis` 的关系

| 维度 | etf-analysis（理论框架） | etf-operation（实战操作·本文件） |
|------|------------------------|--------------------------------|
| 性质 | 工具书，查指标用 | **操作手册，照着做** |
| 内容 | ETF分类、费率、跟踪误差等静态知识 | 盘前流程、盘中修正、经验教训等动态积累 |
| 更新频率 | 低频（产品/规则变化才更新） | **高频（每次操作后补充）** |
| 使用场景 | 分析数据时查阅 | 每次写报告前核对、每次偏差后更新 |

### 1.2 核心原则

```
1️⃣ 框架服务于实战，不是实战服务于框架
2️⃣ 承认预测的局限性，建立纠错机制而非追求"零偏差"
3️⃣ 每发现一个盲区，就补一条规则——体系是迭代出来的
4️⃣ "这次不一样"往往是亏钱的开始，记录历史经验以对抗"这次不一样"的幻觉
```

## 二、盘前分析标准流程（Checklist）

> **每次写盘前报告前，逐条核对以下15项。**

### Step 1：隔夜全球市场扫描

```
⌨️ 执行顺序：

□ 1.1 美股三大指数（道指/纳指/标普500）涨跌及原因
    - 用什么查：akshare stock_us_spot_em 或 Playwright 新浪财经美股板块
    - 重点看：科技股（英伟达/苹果/微软）和半导体（费城半导体指数）
    
□ 1.2 欧洲主要市场（德国DAX/法国CAC）—— 可简略
□ 1.3 亚太市场（韩国KOSPI/日经225/恒指夜期）
    - 特别注意：韩股半导体板块（SK海力士/三星）—— 与A股半导体强相关

□ 1.4 大宗商品
    - 原油（WTI/布伦特）：地缘冲突和通胀预期
    - 黄金：避险情绪和美联储预期
    - 铜博士：全球经济预期

□ 1.5 汇率（美元指数/人民币汇率）
    - 美元走强→新兴市场承压→北向资金流出
    - 人民币贬值→港股承压
```

### Step 2：中国政策与经济日历确认

```
□ 2.1 央行公开市场操作（逆回购到期、MLF续作）
    - 去中国人民银行官网或新浪财经查
    - 影响：流动性态度判断

□ 2.2 重大经济数据发布（官网确认时间和预期值）
    - ⚠️ 7/15教训：必须到国家统计局/央行官网确认发布时间，不要靠记忆！
    - 常见数据：PMI、CPI/PPI、社融、工业增加值、GDP、社零

□ 2.3 新股/IPO日历
    - ⚠️ 7/15遗漏教训：必须查询未来3日新股申购/上市计划
    - 重点关注：融资规模>50亿的大型IPO
    - 查什么：`ak.stock_ipo_declare_em()` ⭐已验证可用
    - 怎么查：`df = ak.stock_ipo_declare_em()` 查看审核状态为"注册生效"的新股
    - 重点关注：融资规模>50亿的大型IPO，关注申购日（资金压力最大的一天）
    - 为什么重要：大型IPO会对同板块存量股产生"资金虹吸"和"替代逻辑"双重冲击
    - 历史参照：2020中芯国际IPO→半导体板块见顶回调3个月；2026长鑫IPO→半导体当日-3.59%

□ 2.4 晚间/当日上午重要新闻
    - 国务院常务会议、部委政策文件、产业新闻
    - 上市公司重大公告（业绩预告、并购重组、IPO进展）
```

### Step 3：ETF数据采集与校验

```
□ 3.1 A股ETF行情获取（主用Sina API）
    - 命令：ak.fund_etf_hist_sina(symbol="sh512480")
    - ⚠️ 注意：Sina数据延迟15分钟，开盘不久的数据可能不准
    
□ 3.2 港股ETF行情获取
    - 同上，前缀sz（深市挂牌的港股ETF）
    
□ 3.3 数据校验——"三看"原则
    ├── 看上期报告数据是否一致（上次收盘价是否对得上）
    ├── 看异常值（单日涨跌幅>15%或<-15%的要核实）
    └── 看成交额是否合理（突然爆量或极度缩量需要关注）

□ 3.4 个股补充数据（当需要分析龙头分化时）
    - 命令：ak.stock_zh_a_hist(symbol="002594", ...)
    - 注意：若akshare失败，使用 stock_zh_a_spot_em（EastMoney实时）
```

### Step 4：盘前预测框架确认

```
□ 4.1 当前市场阶段定位
    ├── 趋势市（单边上涨/下跌）
    ├── 震荡市
    ├── 风格切换期（科技→防御 / 成长→价值 等）
    └── 事件驱动期（地缘冲突、政策出台、财报季）

□ 4.2 本周主要矛盾（用一句话说清楚）
    例："芯片见顶+美伊冲突+长鑫IPO三重压制下，短期维持防御配置。"

□ 4.3 各板块预判 → 填表

| 板块 | 代码 | 盘前判断 | 策略 | 风险等级 |
|------|------|---------|------|:--------:|
|      |      |         |      |          |

□ 4.4 极端情景预案
    ├── 如果数据超预期 → 怎么做
    ├── 如果数据不及预期 → 怎么做
    └── 如果出现黑天鹅（冲突升级/突发政策）→ 怎么做
```

### 3.4 实战经验：为什么EastMoney总被拦截？

```
根本原因（非代码问题，是网络问题）：
  
  本机网络 → [代理/防火墙] → EastMoney API服务器
                                    ↑
                             请求从代理IP发出
                             被EastMoney识别为非正常访问
                              → 连接重置 / 超时 / 502

  只有少数请求从本地直连能通过：
    fund_etf_spot_em()          → 用另一个端口（push2）偶尔能通
    stock_fund_flow_big_deal()  → 用不同API路径
    stock_ipo_declare_em()      → 不同域名
    
  解决方案的思路：
    1️⃣ 在本机网络环境下，接受"EastMoney不可靠"的事实
    2️⃣ 建立以 Sina + Playwright 为主的数据获取体系
    3️⃣ EastMoney的API能用的就用，不能用的不纠结
    4️⃣ Playwright可以模拟浏览器直接打开网页获取数据，不受代理限制
```

## 四、标的池管理机制

### 4.1 三层池架构

```
核心池（9只）—— 每日跟踪，出SIGNAL信号
  沪深300、科创50、红利ETF、银行ETF、医疗ETF、消费ETF、
  半导体ETF、恒生ETF、恒生科技ETF

  黄金ETF、军工ETF、创业板ETF、券商ETF、新能源ETF、游戏ETF

可转债附属池（约8只）—— 日报末尾轻量参考
```

### 4.2 池子调整规则

```
季度审视（3/6/9/12月第一个周末）：
  核心池中连续一季无信号品种降级到观察池
  观察池中持续出现机会的品种升级到核心池

临时审视（触发式）：
  某品种连续3日成交额进全市场前20但不在池中
  重大政策催生新板块
  用户主动要求关注新品种
```

## 五、盘中修正协议

> **盘前预测100%准确是不可能的。关键在于：发现偏差后多快纠正，以及每次纠正后学到的经验是否被记录下来。**
>
> **特别注意：用户根据日报手动操作真实资金，盘中修正的及时性直接影响用户真实盈亏。**

### 4.1 盘中修正时间点

当前设置：12:30（已部署到策略的 afternoon_run）
- 中午12:00~12:30 获取盘中数据，12:30输出盘中修正信号
- 如有重大偏差，用户可在13:00开盘前更新SIGNAL
- 盘中修正不是每天必须的——仅在早盘信号出现方向性重大偏差时触发

### 4.2 让用户可自行判断信号是否有效的"失效条件"

每个信号应附带盘中可对照的失效条件，使用户不等12:30也能自行判断：

```
日报SIGNAL格式示例（从7/18起实施）：
  红利ETF: 信号=0, 置信度=0.7
    失效条件: 如果今日收盘跌破3.10(-2%+), 防御逻辑可能瓦解
  医疗ETF: 信号=0, 置信度=0.5
    失效条件: 如果今日反弹收回0.33以上且放量, 下周可恢复+1
  科创50: 信号=0, 置信度=0.5
    失效条件: 如果今日继续暴跌超-5%+, 可能触发硬止损
```

### 4.3 偏差触发信号

| 信号类型 | 具体表现 | 触发行动 |
|---------|---------|---------|
| **方向性偏差** | 预测某板块上涨但实际下跌>2%（或反之） | 立即盘中修正，发出V2报告 |
| **突发事件** | 盘中出现盘前未覆盖的重大新闻（如地缘冲突升级/政策出台） | 评估影响，必要时发布紧急提醒 |
| **数据源偏差** | 盘前用的数据是前一天的，盘中发现走势完全基于过期数据 | 标注"数据延迟"风险 |

### 4.2 V2（盘中修正版）发布标准

```
何时发布盘中修正版：
  ↳ 当盘前预测与盘中走势出现 2个以上板块的重大方向性偏差时
  ↳ 当出现盘前完全未覆盖的重大事件（如长鑫IPO）
  ↳ 当盘前依赖的关键数据（经济数据时间/预期值）出现错误

V2报告必须包含：
  1. 偏差声明——哪些错了，错在哪里
  2. 偏差原因分析——为什么错（不要找借口，要归因到具体缺失的信息或逻辑漏洞）
  3. 修正后的判断——现在怎么看
  4. 新增的操作建议
  5. 经验教训——补充到本skill的"常见偏差记录"部分
```

### 4.3 盘中分析的注意事项

- 10:00~10:30的走势往往最真实（开盘前30分钟有很多噪音单和程序化交易）
- 14:00~14:30是"聪明钱"的博弈窗口（尾盘前最后决策）
- 14:57~15:00集合竞价期可能有大单异动
- 港股尾盘15:55~16:00也有类似规律
